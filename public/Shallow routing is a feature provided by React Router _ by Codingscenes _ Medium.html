Shallow routing is a feature provided by React Router

Shallow routing is a feature provided by React Router that allows you to change the URL of a page without actually navigating to a new page. This means that the page component is not reloaded, and any state or props that were previously set are preserved.

Shallow routing is achieved using the history object, which is provided by React Router. The history object has a push method that can be used to change the URL of the current page without reloading it. By default, the push method performs a full page reload, but you can enable shallow routing by passing the state object with a shallow property set to true.

Here’s an example of how to use shallow routing in React:

import { useHistory } from 'react-router-dom';

function MyComponent() {
  const history = useHistory();

  function handleClick() {
    history.push('/my-page', { shallow: true });
  }

  return (
    <button onClick={handleClick}>Go to My Page</button>
  );
}

To use shallow routing in React Router, you can use the history object provided by the useHistory hook to push a new URL onto the history stack without actually navigating to a new page. This means that the page component is not reloaded, and any state or props that were previously set are preserved.

In this example, the MyComponent component uses the useHistory hook to get access to the history object. The handleClick function is called when the button is clicked, and it uses the push method of the history object to change the URL of the current page to /my-page using shallow routing.

Note that shallow routing only works for same-page URL changes. If you want to navigate to a different page, you will need to perform a full page reload.

Difference between shallow routing and regular routing in React Router

The main difference between shallow routing and regular routing in React Router is that shallow routing allows you to change the URL of a page without actually navigating to a new page, while regular routing performs a full page reload when the URL changes.

With regular routing, when the URL changes, the entire page is reloaded and the new page component is rendered. This can be slow and inefficient, especially if the new page component is similar to the previous one and only a small part of the page needs to be updated.

With shallow routing, when the URL changes, the page component is not reloaded and any state or props that were previously set are preserved. This means that only the part of the page that needs to be updated is re-rendered, which can be much faster and more efficient.

Shallow routing is useful when you want to change the URL of a page without actually navigating to a new page, such as when you want to update the URL to reflect a change in the page state or to support browser history navigation. Regular routing is useful when you want to navigate to a completely new page, such as when you click on a link or enter a new URL in the address bar.

In React Router, you can use the history object to perform both shallow routing and regular routing. The push method of the history object performs regular routing by default, but you can enable shallow routing by passing the state object with a shallow property set to true.
